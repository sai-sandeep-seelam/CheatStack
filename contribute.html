Contribution Guidelines

Format & Structure

Follow our standard format with clear sections, examples, and explanations. Use markdown for formatting code blocks and sections.

Content Quality

Focus on practical, concise information. Include common use cases, best practices, and gotchas. Avoid lengthy explanations.

Code Examples

Provide working, tested code examples. Include comments where helpful. Follow language-specific style guides.

Metadata

Include proper metadata: title, description, category, tags, and your name as contributor.

Frequently Asked Questions

How long does the review process take?

We typically review submissions within 3-5 business days. For minor improvements or fixes, the process may be faster. You'll receive an email notification when your submission is reviewed.

What format should I use for code examples?

Use markdown code blocks with language specification for syntax highlighting. For example:

```javascript
function example() {
  return "Hello World";
}
```

Will I be credited for my contribution?

Yes! All contributors are credited on the cheatsheet they contributed to. If you provided your GitHub username, we'll also link to your profile.

Can I update my submission after it's published?

Yes, you can submit improvements to your own cheatsheets or any other published cheatsheet using the "Improve Existing" form.
